🛡️ Access Control

Understanding User Roles

Folonite uses role-based access control to ensure users only see and modify what they're authorized for.

🛡️
ADMIN

Admin

Full system access with complete control over users, settings, and configurations.

  • Create and manage users
  • Configure system settings
  • Access all modules
  • Generate all reports
  • Manage approvals
👥
MANAGER

Manager

Department-level oversight with audit and reporting capabilities.

  • Manage assigned properties
  • Run audit sessions
  • Generate reports
  • Approve changes
  • View team assets
📦
USER

User

Operational access for day-to-day asset management tasks.

  • Add and edit assets
  • Generate QR codes
  • Create tickets
  • View assigned assets
  • Scan QR codes

Permission Scoping

Beyond role-based access, Folonite supports property and department-level scoping. This means a Manager might only see assets for specific properties they're assigned to, while an Admin sees everything. Users can be granted access to multiple properties and departments as needed.

💡 Core Concepts

Key Concepts in Folonite

Understanding these fundamental concepts will help you work more effectively with the system.

📦

Assets

Physical items being tracked with rich metadata including status, ownership, procurement details, and lifecycle information.

🏢

Properties

Physical locations or facilities where assets are stored. Assets are organized by property for easier management and auditing.

👥

Departments

Organizational units within properties that help scope access and organize assets by functional areas.

🔳

QR Codes

Physical labels generated for each asset that enable quick identification and mobile scanning for audits and lookups.

🎫

Tickets

Maintenance and support requests that track issues from creation through resolution with full comment history.

✓

Audits

Compliance verification processes that verify asset existence, condition, and location against the system records.

📊

Quantity Normalization

When you create an asset with quantity greater than 1, Folonite automatically creates individual unit-level records. For example, if you add "Office Chairs — Quantity: 5", the system generates 5 unique asset IDs (AST-001 through AST-005), each with its own QR code and tracking history.

✓ This enables precise audit verification and individual unit tracking

📱 Mobile Access

PWA Installation for Field Use

Folonite works great on mobile devices. Install it as a Progressive Web App for the best experience.

📲

Install as App

Add Folonite to your home screen for quick access without opening the browser each time.

🔳

QR Scanning

Use your device camera to scan asset QR codes for instant lookup and verification.

✈️

Offline Access

View cached asset lists even without connectivity. Changes sync automatically when back online.

⚡

Field Operations

Perform audits, update asset status, and create tickets directly from mobile devices.

Installing on iOS (iPhone/iPad) ▼
  1. Open Safari and navigate to your Folonite instance
  2. Tap the Share button (square with arrow up)
  3. Scroll down and tap "Add to Home Screen"
  4. Tap "Add" in the top right corner
  5. Folonite will now appear as an app icon on your home screen
Installing on Android ▼
  1. Open Chrome and navigate to your Folonite instance
  2. Tap the menu button (three dots) in the top right
  3. Tap "Add to Home screen" or "Install app"
  4. Tap "Add" or "Install" to confirm
  5. Folonite will now appear as an app icon in your app drawer
Offline Capabilities ▼

Folonite supports limited offline functionality for field operations:

  • View cached asset lists from your last session
  • Scan QR codes and record verification status
  • Queue changes that sync when connectivity returns
  • View property and department information
